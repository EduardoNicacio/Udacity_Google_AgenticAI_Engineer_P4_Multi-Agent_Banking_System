# Environment Setup

## Project Structure

The workspace is organized into three agent directories, each representing a specialized banking agent:

- **deposit**/: The deposit account agent
  - `agent.py`: Define and configure the deposit agent
  - `agent-card.json`: Agent card with name, description, and skills
  - `agent-prompt.txt`: System instructions for the agent
  - `tools.yaml`: MCP Toolbox configuration for database connectivity

- **loan**/: The loan account agent
  - `agent.py`: Define and configure the loan agent with sub-agents
  - `loan.py`: Implement loan approval workflow using SequentialAgent and ParallelAgent
  - `agent-card.json`: Agent card with name, description, and skills
  - `tools.yaml`: MCP Toolbox configuration for database connectivity
  - `*-prompt.txt`: Various prompt files for sub-agents

- **manager**/: The manager agent that routes requests
  - `agent.py`: Define the manager agent with RemoteA2aAgent connections
  - `agent-card.json`: Agent card with name, description, and skills
  - `agent-prompt.txt`: System instructions for routing logic

- **docs/**: Supplementary materials
  - `deposit.sql`: Database schema and sample data for deposit accounts
  - `loan.sql`: Database schema and sample data for loan accounts
  - `loan-policy.pdf`: Loan policy guidelines (upload to GCS)
  - `loan-customer-info.pdf`: Customer profile information (upload to GCS)

- **requirements.txt**: Python dependencies including come pre-installed:
- `google-adk`: Agent Development Kit for building agents
- `a2a-sdk`: Agent-to-Agent communication protocol
- `toolbox-core`: MCP Toolbox client for database connectivity

- **.env-sample**: Template for environment variables (copy to .env and configure)

## Local Machine Instructions

These instructions will guide you through setting up the multi-agent banking system on your local development machine. Make sure you complete each step before moving to the next.

### Prerequisites

- Python 3.10 or higher installed on your machine
- Google Cloud SDK installed and configured with your credentials

#### Setup Steps

1. Clone the repository and navigate to the project folder

    After cloning, change into the starter directory where you'll be doing all your work:

    ```sh
    cd project 
    ```

2. Create and activate a virtual environment

    It's recommended to use a virtual environment to isolate your project dependencies:

    ```sh
    python -m venv .venv
    source .venv/bin/activate  # On Windows: .venv\Scripts\activate
    ```

    You should see (.venv) in your terminal prompt when activated.

3. Install dependencies

    Install all required Python packages using the provided requirements file:

    ```sh
    pip install -r requirements.txt
    ```

4. Configure environment variables

    Copy the sample environment file and update it with your specific values:

    ```sh
    cp .env-sample .env
    ```

    Open the `.env` file in your editor and configure the following:

    - `GOOGLE_GENAI_USE_VERTEXAI` - Set to TRUE to use Vertex AI (already set)
    - `GOOGLE_CLOUD_PROJECT` - Your GCP project ID
    - `GOOGLE_CLOUD_LOCATION` - Your GCP region (default: us-central1)
    - `MYSQL_HOST` - Your Cloud SQL instance public IP address
    - `MYSQL_USER` - Database username you created
    - `MYSQL_PASSWORD` - Database password for that user
    - `TOOLBOX_URL` - URL where the MCP Toolbox runs (default: <http://127.0.0.1:5000>)
