## 🚨 Important Notes Before Testing

1. **State Management:** Ensure you clear the session state between tests if running multiple scenarios in one run to avoid conflicting "loan request" states.
2. **PDF Content Assumption:** These queries assume standard banking policies where Debt-to-Equity ratios exist (e.g., 1:1 or similar). The `total_value_agent` logic relies on these values being present in the policy PDF.
3. **Guardrails:** Pay close attention to Query #1 and #13 responses. Ensure they do not output strings like "Your customer rating is X" or "Policy requires Y".
4. **A2A Integration:** Verify that when checking equity (Query #9), the Loan Agent successfully calls the Deposit Agent via A2A without timeout errors.
